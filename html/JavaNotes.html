The Basics

Java is a platform-independent programming language. This means a Java program written once will run anywhere (that you can run java)!
  • This is thanks to the Java Virtual Machine, or JVM, responsible for executing Java code on a device.
  • Machine-specific JVM's take the onus of platform dependence off the code itself.
The source file is the name for the java file as a whole. It's standard practice to only include one class per source file, as the file is named after the class contained within.

The Class & The Object

Java, as an object oriented language, deals heavily with objects.
  • An object represents a "module" of a program, containing state and behavior, represented by variables and methods respectively.
Every java application must have at least one class, and one main method per application.
  • A class holds the methods (functions) and variables necessary for your project's or object's operation.
  • The main method is the entry point of a Java program.
Classes are used as blueprints for creating objects.

Running a Java Program

  1. A java file consists, broadly, of three components: Classes, Methods, and Statements.
  2. Write your java code in the source document, a.k.a the source file.
  3. Compile your code to ensure it will run without forseeable errors. This turns it into Java bytecode.
  4. Run the compiled Java bytecode on the Java Virtual Machine (JVM), the software installed on devices which used to run Java code.
