The Basics

To include CSS in your HTML document via a stylesheet, use a <link> element with an href attribute pointing to your stylesheet.


  • For example:
    • <link rel="stylesheet" href="filename or URL">
    You can include CSS directly in your page, inline, i.e. per element, using the <style> tag. Not useful for applying style to multiple sheets.
  • A CSS selector and the properties defined between its curly brackets is called, altogether, a rule.
  • Elements nested within others will inherit rules from their parent element.
  • To override an inherited rule, create a new rule for the element(s) to be overridden.
    • The most specific rule always wins in this case.
    Use a descendant selector to change properties of certain elements within other elements that would otherwise inherit its parent's rules.
    • div h2 {color:black;} This selects a heading within a div element to apply style to. Make it more specific with a class or id in case theres a chance another div with an h2 in it shouldnt be black.
  • Style multiple elements at a time by creating a class selector. State the element type followed by a dot, followed by your desired class name. Then you can define rules to your heart's content! For example:
    • li.yellow{background-color:yellow;}
    To like to style ALL elements with the yellow class, simply omit the element name(s). Your syntax will then be .class
    • .yellow{background-color:yellow;}
  • Elements can belong to more than one class as well. List all the classes between quotes followed by spaces.
    For example:
    • "yellow centered_text important"
  • These rules range in specificity. Element and class by themselves are less specific than "element.class".
    • The most specific selector wins in terms of rule conflict, such as when rules format the same property differently.
    • If even this fails, the lower the rule is on the stylesheet, the more superiority it has. This list element belongs to the yellow and orange classes, which both only designate background color. The orange rule is lower on the page than the yellow rule.
  • A single specific element can be modified using an ID via the # symbol instead of a dot. This means any element can only have one unique ID.
    • #id-name
  • You can even override stylesheets! The lower they're <link>'ed in the head element, the more precedence they have over a previous sheet, just as with selectors.
  • Use /* */ to comment out text in CSS.
    For example:
    • /* This is a comment. */

Text & Fonts

  • A font family refers to:
    • A group of fonts that share design characteristics, i.e. sans serif, serif, monospace, fantasy...
    • A parent font and its variants, i.e. gothic, gothic bold, gothic italic...
  • To include a custom font, either from the web or from your own server, use the font face rule.
    For example:
    • @font-face {
      font-family: "Name of your Font";
      src: url("relative or absolute url for font of one file type"),
      src: url("another url for font of another file type for compatability"); }
    If your fonts are located in a separate folder, do not neglect to use the .. ascend symbol in your url.
  • Some important font properties to know:
    • font-size: 14px, 15%, 1.5em, medium;
    • font-weight: bold, normal;
    • font-style: italic, oblique;
    • text-decoration: underline, overline, line-through
    • text-align: center, start, justified;
    • line-height: 13px, 95%, 1.9em;
    Line height can be set proportionally according to its own default size by using just a number as the property value.

    em and % are proportional modifiers.

The Box Model

  • CSS treats HTML body elements like layered boxes. This is referred to as the Box Model.


    This box consists of four components:
    • Content
    • Padding
    • Border
    • Margin
    The content area contains, of course, the content contained in the element.
    • width: 600px;
    • text-align: center;
      • You may remember this from the Text & Fonts section. It's responsible for aligning ALL of the content in the content pane, as opposed to just text.
    The padding box provides space between the content and the border. Only its size may be edited.
      \
    • padding: 25px;
    • padding-left: 80%;
    The border box contains the content and padding within a box whose appearance can be customized.
    • border: solid 4px #c45525;
    • border-style: solid, groove, dotted;
    • border-width: thin, 9px;
    • border-color: red, rgb(26%, 12%, 75%), #ff0011;
    The margin box provides space between the border and other elements.

    Adjust padding and margin sizes with:
    • margin: 30%;
    • margin-right: 50px;
    When the bottom and top of an element meet each other, the larger margin prevails.

    Specify a position after the border property to edit a specific section.
    For rounded borders:
    • border-top-left-radius: 3em;
    • border-bottom-right-radius: 0px;
    To effect changes on the top, right, bottom, and left, in that order, of your box discretely its helpful to use one property to set them rather than multiple separate statements.
    • padding: top right bottom ;eft;
    • margin: 12px 0px 25px 0px;
    The dimensions of an element are the sums of the widths and lengths of their boxes, and thus their components thereof.
    It's also recommended not to set the height of your element, lest its contents spill over into other elements under unforeseen browser circumstances.
  • Add a decorative image to an element using the <background image> property.
    • background-image: url(url_goes_here.jpg);
    Use this only when an image does not add function to your page, and is used only for decorative purposes.
    Images used this way, by default, are generated in the top left of the element and repeat in a tiled manner from left to right and top to bottom.
    Address the repeating and positioning of your background image with these properties:
    • background-repeat: no-repeat, repeat-x, inherit;
    • background-position: left, bottom, center, 12px, 43%;

Layout

  • An HTML document displays content according to Flow. Elements of a page are displayed how as they're typed in the document, from top to bottom, end to end.
  • Flow applies to inline elements as well, following the flow of the text, i.e. left to right, top to bottom. Text is treated as if it were a flexible inline element.
  • This arrangement of rows, as it were, can be modified to include columns as well using the float property.
    • The element being "floated" must have a width property defined.
    The element is floated above (quite literally on top of) the proceeding elements to its desired location. Subsequently, the inline elements (remember, text is considered an inline element) are arranged around the floated element.
