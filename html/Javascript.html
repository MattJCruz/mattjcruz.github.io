Trivia

  • Javascript, originally "Mocha", was created originally in 1995 by Brendan Eich at Netscape.
  • JScript, a "dialect" of sorts for Javascript, was released simultaneously as Javascript, meaning two different versions of the language were now on the market.
  • There was no standardized syntax for the language at the time. To remedy this, Javascript 1.1 was submitted to the European Computer Manufacturers Association (ECMA) in 1997. The committee appointed to the task of standardizing Javascript made it vendor-neutral, general-purpose, and cross-platform, thusly dubbed "ECMAscript". This is the raw standardized version of Javascript.

The Basics

Javascript is, at the end of the day, a programming language. Many of its properties are shared with many other languages, and as such those facts will be excluded, i.e. variables, booleans, if statements etc., due to the fact I'm already intimately acquainted with them at this point.
  • Use the <script> element to embed Javascript into your page. Put raw javascript between the tags or add a src attribute linking to your external javascript file.
    • It can go in the <head>, where it'll be evaluated before your page loads, or body, where its executed as the page loads, often used to generate content for your page.
  • Javascript is a scripting language, a programming language that's interpreted or compiled at runtime, as opposed to being compiled before runtime as is usually the case with Object Oriented Programs.
  • Declare variables with the let keyword to avoid accidentally reinstantiating variables, as is apparently prone to happen using the previously standart var keyword. (For the record, I have never personally accidentally tried to instantiate a variable twice in ANY language.)
  • There are no explicitly stated data types. They're determined implicitly and apparently can be changed on the fly when circumstances demand it, referred to as "Dynamic Typing". Perhaps useful in some capacity but seems to lead to bad habits like instantiating variables twice.
  • As with other languages, use camelCase when naming them.
  • Variables can have a datatype of Integer, Floating Point Value, String (of characters), or Boolean. A variable with no value has a value of undefined.
  • Name variables starting with a letter, underscore, or dollar sign, and every subsequent character can be one of the above, a number, or any other symbol excluding +, -, and spaces.
    • Additionally they're case-sensitive. NUM1 and num1 are two different variable names.
    • Avoid using Javascript reserved keywords as variable names.
    • Make your variable names meaningful.
  • Application Programming Interfaces (API's) are used to access additional functionality beyond the scope of a simple program.
  • Strings are surrounded with double quotes or single quotes, as it appears characters do not exist as a data type.
  • A variable with a set value will be considered true in a boolean expression, while an empty one will be considered false.
  • Use console.log to write to the javascript console in your environment.
  • To write to an HTML document in Javascript, use the document.write function. This is not recommended as there are (apparently?) better ways to do this.
  • Look into floating point numbers and finite precision.
  • Remember the % (remainder/modulus) operator
  • Declare an array with:
    var arrayOne = new Array();
    var arrayTwo = [ 25, 54, 321, 54];

    Add values to it using:
    arrayOne[0] = 12;
    Omit the assignment operation to use the value located at that specific index.

    Get the length of your array with its .length property. arrayCount - arrayOne.length;

The Document Object Model (DOM)

  • Java script uses the Document Object Model, or DOM to interact with your HTML File.



    This (upside-down) tree of elements is created by your browser parsing your HTML document. Javascript uses it to manipulate elements and their content dynamically. The DOM is only created when a page is loaded in your browser, and it's done so sequentially. Keep this in mind.

  • Gain access to a specific element according to its ID using the <getElementById> function:
    • document.getElementById("idname");
  • The document is accessed at the "root(s)" of the tree and the specified ID is selected. Now we'll use this function to manipulate the content of the element with the specified ID using the innerHTML property.
    • var text = document.getElementById("idname");
text.innerHTML = "New text Added!";
  • Remember to tell the page to execute Javascript code AFTER the document-object-model is fully created. Javascript is read and executed as soon as the <script> tag is added to the DOM, whether by linked JS file or verbatim code. This means it will be referencing objects that haven't been created yet. 
    • function init(){
        /*Insert your code and variables go here*/                
               }
window.onload = init;
    Alternatively, the <script> tag can be placed after all loadable elements, speeding up runtime* and allowing the DOM to fully load prior to loading your Javascript.

  • Create a new HTML element using the createElement function.

    • var li = document.createElement("li");
    This function creates an element of a type specified in the parameters and returns (a reference to) the element. In this statement the reference is stored in a variable called li.

    *This information has not been independently verified at time of writing.

Variables, Functions, Objects, & Properties

Variables

  • Local variables are ones defined inside a function. Global variables are not.
  • Variables are defined with the var or let keywords. They exist for as long as the page does, i.e. their values are not stored.
  • Shadowing a variable refers to defining a variable within a function that shares a name with a global variable. The value of the variable within the function is discrete from the value of the global variable, i.e. the global variable is in the shadow of the local one.

Functions

  • In cases where functions share the same name, by accident or otherwise, the last incidence of a named function will be used.
  • Functions can be used as values.
    • window.onload="init"

    Here the init function, our onload handler used to execute code once the DOM is fully loaded, is set as the value of the onload property of the window object. I had assumed it was being called, but when i added parentheses to my init it seemed to be called as soon as the page was loaded, no matter what. This seems to be a workaround to it.
  • Additionally, functions can be called anonymously.
    • var f = function(num) { return num + 1; }
    var result = f(1);

    This all suggests more that functions ARE values. With this in mind, we can further compact our handler into one line of code for readability.
    window.onload = function() {
    alert("you rule!"); }

Objects

  • An object is, essentially, a collection of properties, AKA: variables!
  • Create an object by setting the value of a variable to a set of parentheses.
    • var objectExample = {
    firstVar: "String",
    varTwo: 32,
    var3rd: ["firstElement", "2ndElement"]
    }

    Then list each variable (property) you'd like, ending each declaration with a comma rather than a semicolon.

  • You can access a property from an object in different ways.

    Via dot notation:
    • objectExample.var3rd[1] = "12";

    Via string notation:

    var b
    As it stands, this flies in the face of OOP principles, where classes are Just objects themselves. This is in direct opposition to java, where objects are built from classes, and thus reusable. Here in javascript the object is synonymous with its "class" declaration.
