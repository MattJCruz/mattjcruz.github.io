The Basics

HTML is primarily a markup language, not a formatting language. It's meant to demarcate elements for SEO (search engine optimization), which makes your web page more easily seen by search engines. In terms of form vs. function, it should be treated as all function and no form.
  • Browsers may provide formatting, but this varies from browser to browser and should not be depended on. Depend on other technologies for formatting, such as CSS and perhaps even Javascript.
Any HTML document you author should be tightly formatted so everything works as intended. This keeps the document up to current specifications, or "up to spec", as outlined by the World Wide Web Consortium.
  • Browsers may pick up the slack if your document is sloppily written but certain features may not work as intended, if at all.
When you write an HTML document be sure to tell your browser:
  • what kind of document it's working with using<!doctype html> at the very top of the page, outside the <html> tags.
  • what encoding to use for your document with <meta charset="utf-8"> in the <head> element.
  • what language your page is written in (English, most likely) with the lang="en" attribute in your html tag.
And tell your user:
  • what your page is all about with a <title> tag in the <head> element.
Including these important elements will help keep your document up to spec.
  • Anything you add to the document needs to go in either the<head> or <body> element, never directly into the <html> element.
Demarcate meaningful sections of your webpage with the <div> element.
  • Use it like this: <div>This section is dedicated to a specific topic. Here goes the content...</div>
  • When navigating file trees, keep in mind this occurs with reference to the document you're currently viewing. Use ".." to ascend a level in the tree, i.e. to navigate to a higher folder and subsequently down to the one you need using /.

    <a href="../HTMLDocuments/otherpage.html">

    This traverses out of the current folder to the superfolder and down into the HTMLDocuments folder for the desired html sheet.
  • Consistency is key. It's best practice to use lowercase for all of your classes and id's. No spaces.
    • This will make referencing them later easier on you if you don't have to remember whether or not this specific id is all lowercase or that specific one is PascalCase.
  • Use the W3C Validator to ensure your document is up to spec (specification) as defined by HTML5.

Linking

The <a> (anchor) element is used to link external items called for when using an element that requires it. Use the <href>(hypertext reference) attribute to link to the destination url.
  • <a href="otherPage.html">
  • The text between the anchor tags will be highlighted as a link, like the link to the W3C Validator above.
  • Use the target="_blank" attribute to open your link in a new tab.
<img> elements require a link to the desired image, done with the src attribute.
  • <img src="Img.png">
Use an <id> to navigate to a certain part of the page with href.
  • id="idname"
  • <a href="#idname">Click here to go to the idname section!</a>
  • Your ID must be unique to the element you're identifying. Otherwise you'll be trying navigating to two sections at once!
Use the <nav> element to create a menu bar listing other pages on your website.
  • Use an unordered list formatted to display as an inline block, rather than a line-broken vertical list, to create a menu bar. 
<nav>
    <ul class="nav">
        <li class="nav">
            <a href="index.html">Home</a>
        </li>
        <li class="nav">
            <a href="CSSNotes.html">CSS Notes</a>
        </li>
        <li class="nav">
            <a href="Javascript.html">Javascript Notes</a>
        </li>
    </ul>
</nav>

Images and Formatting

Using JPEGs to reduce the size of a photo can make it more economical for a user's computer to display on your webpage.
  • Setting the width and height of your images beforehand in an image editor will provide your users with a more optimized version of your webpage.
  • Remember to save the resized version as a copy of the original; do not overwrite it.
Incorporate alt attributes into all of your photos in case, e.g., your user is using a screen reader or the image fails to load.
  • This, also, is in the interest of keeping your document up to spec.
Style an element with a CSS sheet by giving it a class attribute with a distinct name value in quotes.
  • class="yellow"

Interacting With the User

Forms are used to collect data from your user via the <form> element
  • The action attribute specifies the url to which the form should be submitted.
    • <form action="/submit-url">Form content <form>
The user inputs data with the <input> element.
  • The type attribute is used to set the type of input to text fields, checkboxes, and file selectors.
    • <input type="text">
    • <input type="radio">
    • <input type="checkbox">
    Use the <br> element to create line breaks between inputs, since they're inline elements.
    Add an id to link it with a label.
Label a form with the <label> element. Add a for attribute to link it to its respective input field id. This increases the hit area for checkboxes to include their label, so clicking the label will activate the checkbox. Your input field can be nested inside the label or added alongside. Nesting negates the need for a for attribute.
  • Use the name attribute to label the input/data entered in the field. Date fields with no name attribute are not sent.
  • Add placeholder text that tells the user what to enter in a text field with the placeholder attribute.
  • Make sure your user fills out required fields with the valueless required attribute.
  • By default, a <button> sans attributes in a form will submit your form to the action url and furthermore the database (to be expounded upon later...) We want this behavior to be explicit by setting the type attribute to submit.
  • Give your data a name attribute to store it in the database that holds your user's data.
  • Make checkbox choices mutually exclusive by only allowing them to select data under one name, i.e. only one
  • Use the value attribute to label your data further, as in the case where you have discrete specific options for a field.
  • fieldset groups related fields.
    • The <legend> element provides a caption hinting at what to enter or select in your fieldset.
  • Use radio buttons for inputs with one answer and checkboxes for multiple-answer inputs.
  • Make a selection default with the selected attribute, no value. It IS the value.
