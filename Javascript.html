The Basics

Javascript is, at the end of the day, a programming language. Many of its properties are shared with many other languages, and as such those facts will be excluded, i.e. variables, booleans, if statements etc., due to the fact I'm already intimately acquainted with them at this point.
  • Use the <script> element to embed Javascript into your page. Put raw javascript between the tags or add a src attribute linking to your external javascript file.
    • It can go in the <head>, where it'll be evaluated before your page loads, or body, where its executed as the page loads, often used to generate content for your page.

  • Javascript is a scripting language, a programming language that's interpreted or compiled at runtime, as opposed to being compiled before runtime as is usually the case with Object Oriented Programs.

  • Declare variables with the let keyword to avoid accidentally reinstantiating variables, as is apparently prone to happen using the previously standart var keyword. (For the record, I have never personally accidentally tried to instantiate a variable twice in ANY language.)

  • There are no explicitly stated data types. They're determined implicitly and apparently can be changed on the fly when circumstances demand it, referred to as "Dynamic Typing". Perhaps useful in some capacity but seems to lead to bad habits like instantiating variables twice.

  • As with other languages, use camelCase when naming them.

  • Variables can have a datatype of Integer, Floating Point Value, String (of characters), or Boolean. A variable with no value has a value of undefined.

  • Name variables starting with a letter, underscore, or dollar sign, and every subsequent character can be one of the above, a number, or any other symbol excluding +, -, and spaces.
    • Additionally they're case-sensitive. NUM1 and num1 are two different variable names.
    • Avoid using Javascript reserved keywords as variable names.
    • Make your variable names meaningful.

  • Application Programming Interfaces (API's) are used to access additional functionality beyond the scope of a simple program.

  • Strings are surrounded with double quotes or single quotes, as it appears characters do not exist as a data type.

  • A variable with a set value will be considered true in a boolean expression, while an empty one will be considered false.

  • Use console.log to write to the javascript console in your environment.

  • To write to an HTML document in Javascript, use the document.write function. This is not recommended as there are (apparently?) better ways to do this.

  • Look into floating point numbers and finite precision.

  • Remember the % (remainder/modulus) operator

  • Declare an array with:
    var arrayOne = new Array();
    var arrayTwo = [ 25, 54, 321, 54];

    Add values to it using:
    arrayOne[0] = 12;
    Omit the assignment operation to use the value located at that specific index.

    Get the length of your array with its .length property. arrayCount - arrayOne.length;

The Document Object Model (DOM)

Page Marker: 103
  • Java script uses the Document Object Model, or DOM to interact with your HTML File.



    This (upside-down) tree of elements is created by your browser parsing your HTML document. Javascript uses it to manipulate elements and their content dynamically. The DOM is only created when a page is loaded in your browser, and it's done so sequentially. Keep this in mind.

    Javascript uses this function to gain access to a specific element according to its ID:

    document.getElementById("idname");

    The document is accessed at the "root(s)" of the tree and the specified id is selected. Now we'll use this function to manipulate the content of the element. 
                    var text = document.getElementById("idname");
                text.innerHTML = "New text Added!";
    If you're including this code between your <script>tags in the <head> of your page, remember to tell the page to execute the code AFTER the document-object-model is fully created, since it will be executed as soon as the <script> tag is added to the DOM. 
                    function init(){
                        /*Your code here*/                
                }

                window.onload = init;
