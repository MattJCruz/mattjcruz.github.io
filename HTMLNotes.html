The Basics

  • HTML is a markup language, not a formatting language. It's meant to demarcate elements for SEO (search engine optimization) and accessibility. For all intents and purposes it's all function, no form.
    • Browsers may provide formatting, but this can vary from browser to browser and should not be depended on. Use CSS for formatting.

  • Any HTML document you author should be tightly formatted so everything works as intended. This keeps the document up to current specifications (up to spec) as outlined by the World Wide Web Consortium.
    • Browsers will pick up the slack if your document happens to be sloppily written but certain features may not work as intended, if at all.

  • When you write an HTML document be sure to tell your browser:
    • what kind of document it's working with using<!doctype html> at the very top of the page
    • what encoding to use for your document with <meta charset="utf-8"> in the <head> element.
    • what language your page is written in with the lang="en" attribute in your html tag.
    And tell your user:
    • what your page is all about with a <title> tag in the <head> element.

  • Anything you add to the document needs to go in either the<head> or <body> element, never directly into the <html> element.
    • Always add a <title> element to you <head> element to keep your document up to spec.

  • Demarcate meaningful sections of your webpage with the div element.
    • <div>

  • When navigating file trees use ".." to ascend a level in the tree, to navigate to a higher folder and subsequently down to the one you need.
    • <a href="../HTMLDocuments/otherpage.html">

  • Consistency is key. It's best practice to use lowercase for all of your classes and id's. No spaces.
    • This will make referencing them later easier on you if you don't have to remember whether or not this specific id is all lowercase or that specific one is PascalCase.

  • Use the W3C Validator to ensure your document is up to spec (specification) as defined by HTML5.


Linking

  • The <href> (hypertext reference) attribute is used to link external items called for when using an element that requires it.
    • <img src="Img.png">

  • To link to another html document, i.e. webpage, use the <a> (anchor) element with the href attribute with the destination url.
    • <a href="otherPage.html>
    • The text between the anchor tags will be highlighted as a link, like the link to the W3C Validator above.

  • Use an <id> to navigate to a certain part of the page with href.
    • id="idname"
    • <a>href=</a>

  • Use the <nav> element to create a menu bar to help organize your pages.

Images and Formatting

  • Using JPEGs to reduce the size of a photo can make it more economical for a user's computer to display on your webpage.

  • Setting the width and height of your images beforehand in an image editor will provide your users with a more optimized version of your webpage.
    • Remember to save the resized version as a copy of the original; do not overwrite it.

  • Incorporate alt attributes in all of your photos in case, e.g., your user is using a screen reader or the image fails to load.
    • This is in the interest of keeping your document up to spec.

  • Style an element with a CSS sheet by giving it a class attribute with a distinct name value in quotes.
    For example:
    • class="yellow"
